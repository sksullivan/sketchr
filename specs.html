Technical Specifications for Sketchr

Necessary Functionality

  • Indicate North
  • Object labeling
    • Automatic numbers for involved vehicles?
    • Street names
  • JSON data model
  • Local HTML/JS code for webviews
  • Camera
  • Send functionality
  • Delete/undo

Objects to include (in increasing order of placement priority)

  • Roads
  • Involved vehicles (red)
  • Uninvolved vehicles (grey)
  • Road signs
  • Obstacles
  • Stoplights
  • Motion paths
  • Name and number labels

Additional Functionality

  • Step-by-step wizard
  • Accomodate irregular roads
    • Grid-like placement?
    • Preset intersections?
  • Free form paths
  • MongoDB implementation
    • Saves of JSON and pictures
    • Stores geographic metadata that can be queried for

Selling Points (why we're better)

  • Mobile apps for the field
  • Cross platform
  • No internet requirement (RESTful--JSON HTTP POST)
  • OOP enables simple and effective JSON
  • Additional utility--camera, GPS, voice
  • Reconstruction from JSON
  • MongoDB GeoJSON Objects allow geospatial indexes and queries
